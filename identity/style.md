# Style — 코드·커뮤니케이션 스타일

## 커밋 메시지

- Conventional Commits + 영어: `feat:`, `fix:`, `refactor:`, `chore:` 등
- 제목은 변경의 "왜"가 드러나게. `step2` 같은 모호한 메시지 금지

## 커뮤니케이션

- 한국어 + 기술용어는 영어 그대로 (예: "이 컴포넌트 memo 처리하면 리렌더 줄어요")
- 존댓체 기본
- 문서도 같은 패턴
