# Principles — 기술 판단 기준

> 참고용 취향. 기술 선택·코딩 스타일은 에이전트가 best practice를 따르면 됨 — 핵심은 [[sense]] (`sense.md`). 아래는 충돌 시 기준 정도.

## 코드 리뷰에서 막는 것 (반드시 고치게 함)

- **타입 구멍**: `any` 남용, 근거 없는 `as` 캐스팅, `@ts-ignore`로 타입 우회
- **불필요한 리렌더/성능 문제**: 인라인 객체 prop, 무분별한 `useEffect`, 거대 컴포넌트로 인한 리렌더 전파
- **컴포넌트 설계 깨짐**: 관심사 분리 안 됨, 심한 props drilling, 재사용 불가 구조

## 상태관리

- 서버 상태는 TanStack Query — fetch 결과를 전역 스토어에 넣지 않는다
- 전역 상태는 최소화 (zustand/jotai). 진짜 전역만 전역으로, 나머지는 로컬 state
- 가벼운 공유는 Context도 사용 — 라이브러리가 항상 답은 아님

## 스타일링

- Tailwind 기본
- 디자인시스템 컴포넌트 있으면 그걸 우선 — 직접 스타일링은 디자인시스템에 없을 때만

## 추상화

- **3번 반복 후 추출** (rule of three). 두 번까지는 복붙 허용
- 에이전트는 공통 컴포넌트/훅을 선제적으로 빼내지 말 것

## 테스트

- 핵심 비즈니스 로직·유틸만 유닛 테스트
- UI 컴포넌트 테스트는 선별적. 커버리지 숫자 좇지 않음
- 나머지는 타입 + 리뷰로 커버
